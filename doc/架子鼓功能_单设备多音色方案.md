## 二、方案对比与选型

### 2.1 ❌ 排除方案

| 方案 | 排除原因 |
|------|---------|
| **动态加载 ogg** | 每次触摸都需从 Flash 读 + 解码（>50ms 延迟） |
| **SD 卡按需读** | 无 SD 卡 |
| **MP3 格式** | 复杂度高，无硬件解码器 |
| **ML_SynthTools 直接移植** | GPL-3.0 许可证与项目 MIT 不兼容 |

### 2.2 ✅ 候选路径

| 方案 | 资源占用 | 延迟 | 复杂度 |
|------|---------|------|-------|
| **A. 预解码 PCM 驻 PSRAM** | 8 鼓 × 50KB = 400KB PSRAM | < 5ms | 🟢 低 |
| **B. OGG Opus 驻 Flash** | 8 鼓 × 30KB = 240KB Flash | 10-30ms | 🟡 中（需解码） |
| **C. ADPCM 压缩驻 PSRAM** | 8 鼓 × 12KB = 96KB | 5-10ms | 🟡 中 |
| **D. 实时合成（osc+noise+env）** | < 1KB | < 1ms | 🟡 中-高（音质一般） |

### 2.3 推荐方案：**A+B 组合**

- **B**：样本以 OGG 格式存 Flash（压缩率高）
- **A**：首次播放时解压到 PSRAM 缓存，后续直接 PCM 播放
- 兼顾 **存储效率**（Flash 小）+ **响应速度**（PSRAM 快）

### 4.3 PCM 缓存机制（关键性能优化）

**设计要点**：
- 首次触摸某鼓件 → 从 Flash 读 ogg → 解码为 PCM → 缓存到 PSRAM
- 后续触摸 → 直接从 PSRAM 取 PCM → 入队播放
- 缓存满时采用 **LRU 淘汰** 或简单**固定全部缓存**（8 × 50KB = 400KB < 2MB）

```cpp
// main/drum/drum_synth.h (伪代码)
class DrumSynth {
public:
    enum class Piece : uint8_t {
        KICK = 0, SNARE, HIHAT_C, HIHAT_O,
        TOM_HI, TOM_MID, CRASH, RIDE, COUNT
    };

    void Init();   // 初始化 8 个 ogg 数据指针
    void Trigger(Piece p, uint8_t velocity = 100);
    void Stop();
    bool IsPlaying() const;
    void EnterMode();  // 切到架子鼓模式（停止语音监听）
    void ExitMode();   // 退出架子鼓模式（恢复语音监听）

private:
    struct DrumSlot {
        const uint8_t* ogg_data;  // Flash 中 ogg 字节
        size_t ogg_size;
        std::vector<int16_t>* pcm_cache = nullptr;  // PSRAM 缓存
    };
    std::array<DrumSlot, 8> slots_;

    // 懒加载：首次触发时解码
    std::vector<int16_t>* GetOrDecodePcm(Piece p);

    // 入队播放
    void EnqueuePcm(Piece p, uint8_t velocity);
};
```

### 4.4 多音同时播放（Polyphony）

**问题**：用户快速连击 3 个鼓件，需要 3 个音同时叠加播放

**解决方案**：复用现有 `audio_playback_queue_`
- 每个 AudioTask 持有自己的 PCM buffer
- AudioOutputTask 串行从队列取出 → 写入 codec（I2S）
- **关键**：单次 OutputData 内**累加所有正在播放的 PCM 样本**

**实现复杂度**：
- **简单版**（推荐）：每触发一鼓 → 入队一个 PCM 任务 → 任务之间自动串行播放
  - 缺点：8 个鼓**不能严格同时**，但实际上 ESP-IDF 队列时延 < 5ms，人耳几乎无感
- **进阶版**：维护"活跃鼓件表"，每次 codec 写入时混合所有活跃样本
  - 优点：真正的同步
  - 缺点：复杂度高

**推荐先实现简单版**，实测延迟。

### 4.5 触摸区域映射

**圆形屏 360×360 几何设计**：

```
中心 (180, 180)
     ╭─────────╮
   ╱             ╲
  │   ┌─ 0: Kick ┐ │   ← 顶（12 点方向）
  │ ┌─┴─────────┴─┐│
  │ │  1: Snare   ││
  │ └─┬─────────┬─┘│
  │   │ Hi-hat   │ │   ← 3 点钟 / 9 点钟
  │ ┌─┴─────────┴─┐│
  │ │  Crash/Ride │ │
  │ └─┬─────────┬─┘│
   ╲ │  Toms    │ ╱
     ╰─────────╯
```

**8 扇区划分**（极坐标）：
- 每个扇区占 45°（360° / 8）
- 扇区编号：0=正上（12 点），1=右上，2=右，3=右下，4=下，5=左下，6=左，7=左上
- 触摸 (x, y) → 中心偏移 → atan2(dy, dx) → 扇区索引
- 中心半径内（R < 30）→ 默认触发 Kick（底鼓最常用）

**实现要点**：
```cpp
// 触摸坐标 → 鼓件 ID
DrumPiece TouchToPiece(int16_t x, int16_t y) {
    int16_t cx = 180, cy = 180;
    int16_t dx = x - cx, dy = y - cy;
    float dist = sqrt(dx*dx + dy*dy);
    if (dist < 30) return KICK;  // 中心 = Kick

    float angle = atan2(dy, dx) * 180.0f / M_PI;  // -180 ~ 180
    if (angle < 0) angle += 360.0f;

    int sector = (int)((angle + 22.5f) / 45.0f) % 8;
    return drum_pieces[sector];
}
```

**切换流程**：
```
[Hidden 鱼眼]
  ↓ 触摸第一层
[Menu 学业子菜单] (时钟 + 架子鼓图标)
  ↓ 触摸架子鼓图标
  ├─ 当前状态是 Idle → 进入 [DrumPad 架子鼓模式]
  └─ 当前是 Listening/Speaking → 提示 + 忽略
  ↓ 触摸非鼓区域 / 长按 3s
  └─ 退出 → [Hidden 鱼眼]
```

**与语音系统冲突解决**：
```cpp
void AttitudeDisplay::EnterDrumPad() {
    if (Application::GetInstance().GetDeviceState() != kDeviceStateIdle) {
        ShowNotification("忙碌中，稍后再试", 2000);
        return;
    }

    // 关闭唤醒词检测（避免触摸误触）
    audio_service_.EnableWakeWordDetection(false);
    study_sub_state_ = StudySubState::DrumPad;
    ShowDrumPadUI();
}

void AttitudeDisplay::ExitDrumPad() {
    audio_service_.EnableWakeWordDetection(true);
    study_sub_state_ = StudySubState::Hidden;
    HideDrumPadUI();
}
```

## 五、关键资源计算

### 5.1 Flash 占用

```
8 个 ogg 样本 × 平均 25KB = 200KB
```

**对照**：
- 当前项目 Flash 占用 ~3MB
- 200KB 占增加 6.7% → ✅ 可接受

### 5.2 PSRAM 占用

```
缓存策略 A：全量缓存所有 8 个 PCM
  8 × 50KB PCM = 400KB PSRAM

缓存策略 B：懒加载（首次触发才缓存）
  首次：0KB；连续使用：逐渐填满
  实际使用：~200-400KB

PSRAM 总共 2MB → 占用 20% → ✅ 充足
```

### 5.3 RAM 占用

```
8 个 DrumSlot（指针 + 缓存指针）: 8 × 16B = 128B
当前正在播放的 PCM 临时变量: < 1KB
LVGL 触摸 + 动画: ~2KB
总计: < 5KB RAM
```

### 5.4 性能估算

| 操作 | 耗时 | 用户感知 |
|------|------|---------|
| 触摸 → 触发 (有缓存) | < 10ms | 几乎无感 |
| 触摸 → 触发 (首次, 需解码) | 30-50ms | 短暂延迟 |
| OGG → PCM 解码（首次）| 20-40ms / 样本 | 一次性 |
| 多音同时播放 | 各 5ms 间隔 | 几乎同步 |

## 六、关键风险与缓解

| 风险 | 概率 | 影响 | 缓解 |
|------|------|------|------|
| **小喇叭低音差** | 🟡 高 | kick 听不出 | 接受现状 / 用 EQ 强化 |
| **OGG 解码内存抖动** | 🟡 中 | 偶尔卡顿 | 全量 PSRAM 缓存 |
| **触摸延迟 > 30ms** | 🟢 低 | 手感差 | LVGL 触摸 + 异步解码 |
| **唤醒词误触发** | 🟡 中 | 模式被退出 | 架子鼓模式禁用 wake word |
| **采样率不匹配** | 🟢 低 | 音调失真 | 统一 16kHz 制作 |
| **PSRAM 不足** | 🟢 低 | 缓存失败 | 懒加载 + LRU 淘汰 |
| **触摸热区冲突** | 🟡 中 | 误触邻区 | 扇区留 5° 死区 |

## 八、关键决策点

| 决策点 | 推荐选择 | 理由 |
|--------|---------|------|
| **样本格式** | OGG Opus 16kHz mono | 复用 PlaySound 机制 |
| **存储位置** | Flash 静态嵌入 | 无 SD 卡，启动即可用 |
| **缓存策略** | 懒加载 + 全量缓存到 PSRAM | 首次慢后续快，2MB PSRAM 充足 |
| **多音同时** | 复用现有 playback_queue_ | 简单可靠，< 5ms 间隔 |
| **触摸分区** | 8 扇区 + 中心 Kick | 圆形屏直观 |
| **进入方式** | 学业菜单 → 架子鼓图标 | 与现有 UI 一致 |
| **退出方式** | 触摸中心 / 长按 3s / 唤醒词 | 多种退出方式 |
| **与语音冲突** | 架子鼓模式禁用 wake word | 避免误触发 |
